# 个人消费统计网站 - 开发说明文档

> 版本：v1.0 | 最后更新：2026-06-18

---

## 一、项目概述

### 1.1 项目目标
开发一款轻量级个人消费统计网站，帮助用户实现快速记账、直观查看消费结构、有效控制预算。核心价值在于**记录快、看盘清、预警灵**。

### 1.2 技术栈
| 层级     | 技术选型                    | 说明                                     |
| -------- | --------------------------- | ---------------------------------------- |
| 前端     | Vue 3 + Vite + Element Plus | 组合式API，按需加载                      |
| 图表     | ECharts 5                   | 按需引入（饼图、柱状图、折线图、环形图） |
| 状态管理 | Pinia                       | 存储用户信息、主题、当前账本             |
| 网络请求 | Axios                       | 统一拦截器处理Token和错误                |
| 后端     | Spring Boot 3.x             | 配套Spring Security或JWT独立鉴权         |
| ORM      | MyBatis-Plus                | 配合LambdaQueryWrapper简化查询           |
| 数据库   | MySQL 8.0                   | 支持JSON字段（备用扩展）                 |
| AI接口   | DeepSeek API                | 仅VIP用户可调用消费顾问                  |
| 部署     | Docker + Nginx              | 可选，后期容器化部署                     |

## 二、功能需求清单

### 2.1 当日统计页（`/dashboard`）

| 功能模块     | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| 快捷记账表单 | 分类下拉（正餐/零食/外卖/购物/娱乐/交通/人情世故/借出借入/其他）+ 金额输入 + 用途输入框（手动填写）+ 日期选择（默认今日）。购物子类（网购/实体店）联动展示。用途输入框旁预设常用标签（如“工作餐”、“买衣服”、“加油”），点击自动填入，减少打字成本 |
| 预算进度条   | 展示当日/当月预算进度，置于页面顶部。前端根据接口返回百分比渲染：<80%绿色😊，80%~100%橙色😐，>100%红色😱。用户可在“我的”页面设置日限额和月限额（普通用户与VIP共有） |
| 当日饼图     | 展示当日各分类支出金额占比，悬浮显示具体数值                 |

### 2.2 月度/年度统计页（`/statistics`）

| 功能模块       | 说明                                                     |
| -------------- | -------------------------------------------------------- |
| 月度明细表     | 选中月份的所有流水记录，支持按分类筛选和分页             |
| 月度柱状图     | 横轴为当月日期（1~31日），每日支出/收入双柱对比          |
| 年度折线图     | 横轴为1~12月，支出/收入两条趋势线对比                    |
| 收支构成环形图 | 当月总支出 vs 总收入占比，以及支出分类占比（可点击下钻） |
| 支出榜前十     | 当月金额最高的10条记录（展示用途、日期、金额）           |

### 2.3 我的页面（`/profile`）

| 功能模块    | 说明                                                         |
| ----------- | ------------------------------------------------------------ |
| 个人信息    | 展示用户名、邮箱、会员状态及到期时间                         |
| 修改密码    | 旧密码验证 + 新密码确认（BCrypt加密）                        |
| VIP展厅模式 | VIP功能（AI顾问、多账本、永久存档、数据导出）全部展示但处于锁定状态（灰色加锁图标）。用户点击任意锁定功能时，弹出“3天免费体验（无需绑卡）”窗口，后端校验该用户是否已用过体验，若未使用则开通3天VIP并记录，若已用过则引导直接付费 |
| 主题切换    | 亮/暖/暗三色一键切换，偏好保存至后端。实现方式采用CSS Variables，切换时修改`document.documentElement.dataset.theme` |
| 预算设置    | 日限额和月限额配置，保存至`budget_setting`表                 |
| 账本管理    | 下拉切换当前活跃账本（默认账本ID=0），可新建/重命名/删除自定义账本。所有记账和统计接口均需携带`bookId`参数，实现数据隔离 |

### 2.4 AI消费顾问（VIP专属）

| 功能模块 | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| AI对话   | 用户输入消费相关问题（如“我这个月超支了吗？”），后端从数据库拉取用户当月统计摘要（总支出、分类占比、预算执行情况），拼接成结构化Prompt后调用DeepSeek API，返回针对性建议。初期不搭建知识库，采用Prompt工程方式实现，每个VIP用户每日调用上限20次 |

## 三、图表类型决策说明

| 图表     | 类型       | 理由                                                         |
| -------- | ---------- | ------------------------------------------------------------ |
| 月度对比 | **柱状图** | 粒度细（按天），柱体高度差异更醒目，便于快速定位“哪天花冒了” |
| 年度趋势 | **折线图** | 粒度粗（按月），折线最能清晰展现全年升降起伏的节奏           |
| 当日构成 | 饼图       | 直观展示当日钱花在了哪些类别上                               |
| 收支占比 | 环形图     | 比饼图更美观，适合展示支出与收入的整体比例关系               |

## 四、数据库设计（核心表结构）

### 4.1 用户表（`user`）

| 字段            | 类型         | 说明                   |
| --------------- | ------------ | ---------------------- |
| id              | bigint       | PK，自增               |
| username        | varchar(50)  | 唯一                   |
| password        | varchar(100) | BCrypt加密存储         |
| email           | varchar(100) |                        |
| role            | tinyint      | 0=普通，1=VIP          |
| vip_expire_time | datetime     | NULL表示无VIP或已过期  |
| trial_used      | tinyint      | 0=未用过体验，1=已用过 |
| theme           | varchar(10)  | 亮/暖/暗，默认亮       |
| created_at      | datetime     |                        |
| updated_at      | datetime     |                        |

### 4.2 消费记录表（`expense_record`）

| 字段         | 类型          | 说明                                                 |
| ------------ | ------------- | ---------------------------------------------------- |
| id           | bigint        | PK，自增                                             |
| user_id      | bigint        | 外键关联user.id                                      |
| book_id      | bigint        | 账本ID，默认0为默认账本。所有查询均需带该字段过滤    |
| category     | varchar(20)   | 正餐/零食/外卖/购物/娱乐/交通/人情世故/借出借入/其他 |
| sub_category | varchar(20)   | 网购/实体店（仅购物时使用），其余为空                |
| amount       | decimal(10,2) |                                                      |
| type         | tinyint       | 0=支出，1=收入                                       |
| purpose      | varchar(100)  | 用户手动填写的用途描述                               |
| record_date  | date          | 消费日期                                             |
| created_at   | datetime      |                                                      |
| updated_at   | datetime      |                                                      |

### 4.3 预算设置表（`budget_setting`）

| 字段         | 类型          | 说明               |
| ------------ | ------------- | ------------------ |
| id           | bigint        | PK，自增           |
| user_id      | bigint        | 外键关联user.id    |
| type         | tinyint       | 0=日预算，1=月预算 |
| limit_amount | decimal(10,2) | 限额               |
| updated_at   | datetime      |                    |

### 4.4 账本表（`book`）

| 字段       | 类型        | 说明                                       |
| ---------- | ----------- | ------------------------------------------ |
| id         | bigint      | PK，自增                                   |
| user_id    | bigint      | 外键关联user.id                            |
| name       | varchar(50) | 账本名称，如“家庭账本”                     |
| is_default | tinyint     | 0=否，1=是（每个用户有且仅有一个默认账本） |
| created_at | datetime    |                                            |

> **说明**：默认账本（book_id=0）无需在book表存记录，系统自动识别。

### 5.5 VIP模块（需认证）

## 六、前端开发规范

### 6.1 目录结构

src/
├── api/ # 接口定义（按模块拆分）
├── assets/ # 静态资源
├── components/ # 公共组件
├── composables/ # 组合式函数（如useTheme, useBudget, useBook）
├── layouts/ # 布局组件（含侧边栏/顶部导航）
├── pages/ # 页面
│ ├── Dashboard/ # 当日统计
│ ├── Statistics/ # 月度/年度统计
│ └── Profile/ # 我的
├── router/ # 路由配置
├── stores/ # Pinia状态（user, theme, book）
├── styles/ # 全局样式（含三套主题变量）
└── utils/ # 工具函数

### 6.2 主题切换实现方式

```
使用CSS Variables定义色值
切换时修改document.documentElement.dataset.theme，并调用/api/user/theme保存偏好。
```



### 6.3 记账表单交互优化

- 用途输入框旁预设常用标签（如“工作餐”、“买衣服”、“加油”），点击自动填入。
- 金额输入框自动聚焦，支持键盘Enter键快速提交表单。
- 提交成功后清空表单并刷新当日饼图。

### 6.4 移动端适配要求

- 页面宽度须自适应手机浏览器，表单采用flex/grid流式布局。
- 侧边栏在移动端默认收起，通过汉堡菜单展开。



## 七、后端开发规范

### 7.1 统一响应格式

{
  "code": 200,
  "message": "success",
  "data": { ... }
}

### 7.2 错误码定义

| 错误码 | 说明                                          |
| :----- | :-------------------------------------------- |
| 401    | Token无效或已过期，前端跳转登录页             |
| 403    | 权限不足（如普通用户调用AI接口或VIP专属功能） |
| 404    | 资源不存在                                    |
| 400    | 参数校验失败                                  |
| 500    | 服务器内部错误                                |

### 7.3 JWT配置

- 过期时间：7天
- 请求头：`Authorization: Bearer <token>`
- 拦截器统一校验，Token过期返回401

### 7.4 AI接口调用规范

- DeepSeek API Key配置在环境变量中，不得硬编码。

- Prompt模板示例：

  你是一位个人财务顾问。用户本月消费数据如下：
  总支出：{totalExpense}元，总收入：{totalIncome}元。
  分类占比：餐饮{ratio}%，购物{ratio}%，娱乐{ratio}%...
  预算执行：日预算{dayBudget}元，日平均{dayAvg}元；月预算{monthBudget}元，已用{monthUsed}元。
  用户问题：{userQuestion}
  请根据以上数据给出具体、可操作的建议，不超过200字。
